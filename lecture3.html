1. File Paths in HTML

A file path tells the browser where to find a file (like an image, CSS, video, or another HTML page). File paths are very important because they decide how your project links to its resources.

There are 2 main types of file paths:
  • Absolute Path ‚Äì the complete URL of the file (e.g., https://example.com/images/logo.png)
  • Relative Path ‚Äì the location of the file relative to your current HTML file

Absolute Path

An absolute path gives the complete location of a file or resource. It usually starts with http://, https://, or the root directory. This path will always point to the same place, no matter where the HTML file is located.

Syntax:

<img src="https://www.example.com/images/picture.jpg" alt="Example Image">

Example:

<img src="https://cdn.britannica.com/16/234216-050-C66F8665/beagle-hound-dog.jpg"
     alt="Beagle Dog">

Output:

Relative Path

A relative path specifies the location of a file relative to the current HTML file. Instead of writing the full URL, you only write the path starting from your project folder. Relative paths are commonly used when files (like images, CSS, or JS) are stored in the same project directory.

Syntax:

<img src="images/picture.jpg" alt="Relative Example">

Example:

<img src="images/sample.jpg" alt="Sample Image">

2. HTML Boilerplate

An HTML Boilerplate is the basic structure of any HTML document. It includes the required tags that every HTML page must have in order to work properly in a browser. Think of it as a starter template for your web pages.

The main parts of an HTML Boilerplate are:
  • <!DOCTYPE html> ‚Äì Defines the document type and HTML version.
  • <html> ‚Äì The root element of the page.
  • <head> ‚Äì Contains metadata (like title, charset, viewport).
  • <title> ‚Äì Sets the title of the page (seen on the browser tab).
  • <body> ‚Äì Holds all the visible content (text, images, links, etc.).

<!DOCTYPE html>

The <!DOCTYPE html> declaration defines the document type and version of HTML. In modern web development, it tells the browser that the page is written in HTML5. It is always written at the very top of the HTML document.

Syntax:

<!DOCTYPE html>

Example:

<!DOCTYPE html>

Output:

(No visible output ‚Äî it just instructs the browser.)

<html>

The <html> tag is the root element of an HTML page. It wraps all the content, including the <head> and <body> sections. Every HTML document starts with this tag.

Syntax:

<html>
   ... content ...
</html>

Example:

<html>
   <head></head>
   <body></body>
</html>

Output:

(No direct output ‚Äî but the page won‚Äôt work without it.)

<head>

The <head> element contains metadata about the document. This includes the <title>, character encoding, viewport settings, and links to CSS or JavaScript. Content inside <head> is not displayed on the page.

Syntax:

<head>
   <title>My Page</title>
</head>

Example:

<head>
   <meta charset="UTF-8">
   <title>Welcome Page</title>
</head>

Output:

(Only the title will show up in the browser tab.)

<title>

The <title> tag sets the title of the webpage, which appears in the browser‚Äôs tab or window. It is always placed inside the <head> element.

Syntax:

<title>Page Title</title>

Example:

<head>
   <title>My First Website</title>
</head>

Output:

(Shows ‚ÄúMy First Website‚Äù in the browser tab.)

<body>

The <body> tag contains all the visible content of a webpage, such as headings, paragraphs, images, links, lists, and tables. Anything inside the <body> is shown in the browser window.

Syntax:

<body>
   ... visible content ...
</body>

Example:

<body>
   <h1>Welcome!</h1>
   <p>This is my first web page.</p>
</body>

Output:

Welcome!

This is my first web page.

2. div, class, and id in HTML

In HTML, <div>, class, and id are used to group and style content. They are very important when designing layouts and applying CSS or JavaScript.

Key points:
  • <div> ‚Äì A generic container (block-level element) used to group elements.
  • class ‚Äì Used to apply the same style to multiple elements.
  • id ‚Äì A unique identifier for a single element.

1. <div> Tag

The <div> tag is a container that groups content together. It doesn‚Äôt have any special meaning by itself but is widely used for layout and design.

Syntax:

<div>
    Content goes here
</div>

Example:

<div>
    <p>This is inside a div</p>
</div>

Output:

This is inside a div

2. class Attribute

The class attribute lets you apply the same CSS style to multiple elements. Classes can be reused across your page.

Syntax:

<tag class="classname">Content</tag>

Example:

<style>
    .highlight { color: red; font-weight: bold; }
</style>

<p class="highlight">This text is red</p>
<div class="highlight">This box text is also red</div>

Output:

This text is red

This box text is also red

3. id Attribute

The id attribute is used to uniquely identify one element on a page. It must be unique and is often used for JavaScript and CSS targeting.

Syntax:

<tag id="uniqueID">Content</tag>

Example:

<style>
    #special { background-color: yellow; padding: 5px; }
</style>

<p id="special">This text has a yellow background</p>

Output:

This text has a yellow background
